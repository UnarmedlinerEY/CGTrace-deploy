Onboarding design · Proposed 30 Jun → shipped 5 Jul

What a new Standard-tier firm clicks through
before they reconcile a single client.

Standard ($79/mo Pro Practice) gets practice-management sync and the per-client "Build cost base history" flow, but not Autopilot's automated multi-client batching. This page was written on 30 Jun as a sequencing proposal; on 5 Jul it shipped in-app as the Build-cost-base worklist — with one deliberate reversal: the push into the return moved from every tier to Premium. The proposal and the pivot are both kept here, because changing a call on new evidence is the job.

Standard tier · Pro Practice
First-run setup, step by step

Seven steps — shipped as designed, except the one we reversed

Steps 1–3 are bulk and firm-level, run once. Step 4 is per-client and repeats once per new client. Steps 5–7 are ongoing — 6 deliberately is not the build flow, and 7 is where the design changed between proposal and ship.

01
Connect practice manager
Firm-level · once · already built
PM sync

Pick Xero Practice Manager or MYOB AE/AO. One bulk pull brings in every client, family group, job and tax-return status the PM holds — no manual re-entry of names, TFNs, or job assignments.

Return jobs & status surface

04
Reconstruct each client, once
Per client · repeats per new client · already built
⚡ Build cost base history

For each synced-but-unbuilt client, the firm runs the AI build once from the worklist: pull that client's own broker history, search a prior-firm workpaper PDF, or fall back to registry data — then review every flagged row before anything commits. One client at a time, by design — this is the Standard-tier ceiling, not a bug. (A pre-filled sample client — the "Build cost base with AI · SAMPLE" walkthrough — shows the full flow end to end without touching real data.)

05
Ongoing client sync
Firm-level · continuous · partly built
PM sync

New clients keep flowing in automatically every time the PM re-syncs — that part already works, it's the same mechanism as step 1. Archiving a client's portfolio when the PM shows them leaving the firm does not exist yet — there's no archived/closed-portfolio concept anywhere in the data model today. Scoped as future work, not built this round.

New clients: automatic Leaver archive: not built
06
Ongoing reconciliation — not the wizard
Per client · repeats every period · already built, different tool
Tracker, not wizard

Pulling in new transactions for a client that already exists happens in Portfolio Tracker's "Start new reconciling period" → Download Transactions (API / Import Document) or Manual mode — not by re-running the build. The build flow reconstructs a client's cost-base history once; every reconciliation period after that flows through the Tracker's five-stage workflow.

⬇ Download transactions Manual mode (Parcel/Sell/Action) ↺ Rollback
07
Push to the return — the step we reversed
Per return job · Premium action · shipped 5 Jul
Changed between proposal and ship

The 30 Jun proposal had the manual "↑ Sync to XPM" push available on every tier, gating only automation. The shipped call is stricter: Standard stops at a built, reviewed cost base plus the audit-trail export — the preparer keys the figure into the return by hand — and the push itself (one-click or Autopilot batch) is the Practice-tier action. That friction is the deliberate upgrade lever, not a limitation we papered over; the Pricing chapter carries the same logic.

Standard: export + hand-key Premium: pmPushCGT
Honest edge

Proposal → shipped, with one reversal on the record. Written 30 Jun as a sequencing proposal; on 5 Jul the sequence shipped as the in-app Build-cost-base worklist (steps 1–4 now chain as one flow). Step 7 shipped opposite to the proposal — push moved to Premium once pricing work showed the export-vs-push friction was the cleanest upgrade lever. The original call is named, not hidden, in the step card above — reversing a documented decision beats quietly rewriting it.

Step 5's leaver-archive is the one genuine gap. Everything else above is existing capability, re-sequenced. Archiving a client's portfolio when they leave the firm has no home in the data model yet and would need real design work, not just UI sequencing.
